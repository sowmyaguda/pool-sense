Welcome to SwimSafe AI, a personalized swim/no-swim safety advisor. Every summer, millions of people swim without knowing if the water chemistry is safe for their health. SwimSafe AI bridges this gap by combining raw pool readings with individual health profiles. 

This project was built from scratch using the Google Antigravity AI Coding Assistant. Antigravity helped me scaffold the project via the Agents CLI, write the ADK 2.0 graph logic, build a custom FastAPI backend, and develop the clean HTML5 dashboard you see today, complete with swimmer profile management and help tooltips.

[0:40 — WHAT IT IS & VALUES]
SwimSafe AI is an ADK 2.0 multi-agent assistant built for the "Agents for Good" track. It turns complex chemical readings into clear, actionable advice, preventing rashes, respiratory flares, and waterborne illnesses by tailoring recommendations to children, seniors, and sensitive groups.

[1:10 — ARCHITECTURE & SECURITY]
(show assets/architecture_diagram_v2.png)
Our architecture diagram shows the end-to-end flow. The input first passes through a Security Checkpoint node. This is a critical security feature: it scrub PII (like emails and phone numbers), stops prompt injection, and restricts negative values before hitting the LLM. 

Once cleared, the Orchestrator delegates tasks to two sub-agents via AgentTool: the Pool Analyzer Agent, which uses a decoupled Model Context Protocol (MCP) server to check guidelines and calculate the Langelier Saturation Index, and the Swimmer Safety Analyst Agent, which handles individual swimmer risks. If conditions are dangerous, a Human-in-the-Loop (HITL) Checkpoint interrupts the graph to ask for confirmation.
