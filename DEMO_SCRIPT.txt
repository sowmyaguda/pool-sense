[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the underlying architecture. The flow starts with user input going into the Security Checkpoint node, which scrubs PII and checks for prompt injection. Once cleared, it moves to the Orchestrator. The Orchestrator delegates tasks to two sub-agents: the Pool Analyzer Agent, which uses our custom MCP server to check chemical guidelines and calculate the Langelier Saturation Index, and the Swimmer Safety Analyst Agent, which checks swimmer profile risks. 

If conditions are extremely dangerous, the HITL Checkpoint node pauses execution to ask the user to confirm they want to override the warning. Finally, the response is rendered in a clean Markdown report.
